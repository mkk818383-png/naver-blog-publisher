# 사용자 수정 이력

이 문서는 사용자가 초안을 고친 내용을 재사용 가능한 규칙으로 누적한다. 가장 최근의 명시적 수정이 이전 수정, 고정 템플릿, 표본 스타일보다 우선한다.

## 기록 형식

각 교정은 반드시 다음 네 필드로 기록한다.

- 기존 초안의 문제
- 사용자가 요구한 수정
- 재사용 가능한 일반 규칙
- 특정 글에만 해당하는 예외

예외가 없으면 `없음`으로 적는다. 특정 글의 사실을 일반 규칙으로 승격하지 않는다.

## 우선순위와 호칭

- `CORRECTION-PRIORITY-01` 최신 사용자 수정 > 누적 사용자 수정 > 승인된 고정 템플릿 > 역할이 제한된 핵심 표본 > 일반 관행 순으로 적용한다.
- `CORRECTION-NAME-01` 동행 남성을 부르는 현재 표현은 `남친`이다. `예랑`과 `예비신랑`은 금지한다.
- 표본, 외부 자료 또는 과거 초안의 표현은 위 규칙을 덮어쓸 수 없다.

## 누적 교정

### CORRECTION-INTRO-01

- 기존 초안의 문제: 업체 특징이나 일반 소개로 글을 시작했다.
- 사용자가 요구한 수정: 그날의 일정과 방문 계기를 먼저 보여 준다.
- 재사용 가능한 일반 규칙: 사용자가 제공한 일정, 동행, 방문 계기 가운데 확인된 내용을 도입의 출발점으로 삼는다.
- 특정 글에만 해당하는 예외: 일정이나 계기를 제공받지 못했다면 만들지 않고 확인된 선택 이유만 짧게 쓴다.

### CORRECTION-RETURN-01

- 기존 초안의 문제: 재방문인데도 소문이나 일반 평가를 먼저 강조했다.
- 사용자가 요구한 수정: 단골 관계와 이전 방문 경험을 먼저 밝힌다.
- 재사용 가능한 일반 규칙: 재방문 사실이 확인되면 일반 평판보다 사용자와 가게의 실제 관계를 우선한다.
- 특정 글에만 해당하는 예외: 첫 방문이거나 이전 경험을 제공받지 않았다면 단골 관계를 추정하지 않는다.

### CORRECTION-FLOW-01

- 기존 초안의 문제: 장소 정보 다음 내용이 실제 방문 순서와 무관하게 배열됐다.
- 사용자가 요구한 수정: 실제 방문 동선에 따라 이야기를 진행한다.
- 재사용 가능한 일반 규칙: 장소 정보 뒤에는 주차 또는 이동, 입장, 메뉴 탐색, 주문, 상차림, 식사의 확인된 순서를 유지한다.
- 특정 글에만 해당하는 예외: 입력에 순서가 없으면 사실을 임의의 체험 순서로 꾸미지 않는다.

### CORRECTION-ORDER-01

- 기존 초안의 문제: 주문 메뉴만 나열하고 선택 이유가 빠졌다.
- 사용자가 요구한 수정: 인원, 배고픔, 앞선 일정 등 개인적인 이유를 넣는다.
- 재사용 가능한 일반 규칙: 제공된 상황이 실제 메뉴 선택에 영향을 주었다면 주문 결과와 함께 설명한다.
- 특정 글에만 해당하는 예외: 선택 이유를 제공받지 않았다면 합리적으로 보이는 이유를 만들어내지 않는다.

### CORRECTION-MEMORY-01

- 기존 초안의 문제: 개인적인 추억과 작은 농담이 사라지거나 일반적인 문구로 바뀌었다.
- 사용자가 요구한 수정: 사용자가 제공한 개인적 맥락을 살린다.
- 재사용 가능한 일반 규칙: 입력에 있는 추억과 농담은 사실관계를 바꾸지 않는 범위에서 본문 흐름에 남긴다.
- 특정 글에만 해당하는 예외: 자료에 없는 추억, 대화, 반응은 만들지 않는다.

### CORRECTION-DRAWBACK-01

- 기존 초안의 문제: 칭찬만 이어지거나 아쉬움이 막연했다.
- 사용자가 요구한 수정: 실제로 겪은 구체적인 아쉬움을 적는다.
- 재사용 가능한 일반 규칙: 아쉬움이 제공되면 대상, 상황, 영향 또는 당시 대응 중 확인된 내용을 구체적으로 쓴다.
- 특정 글에만 해당하는 예외: 아쉬움을 제공받지 않았다면 균형을 맞추기 위해 단점을 발명하지 않는다.

### CORRECTION-AFTER-01

- 기존 초안의 문제: 식사 후 일정이 본문과 분리되거나 하루의 흐름에서 빠졌다.
- 사용자가 요구한 수정: 다음 일정을 하루 이야기의 결말로 연결한다.
- 재사용 가능한 일반 규칙: 확인된 식사 후 일정이 있으면 `식사 후 일정` 구역에서 식사와 다음 동선을 잇는다.
- 특정 글에만 해당하는 예외: 초기 교정에서는 사용자가 경험이 없다고 밝힌 경우에만 생략한다고 표현했으나, 이후 승인된 안전 기준에 따라 실제 식사 후 경험이 제공된 경우에만 구역을 넣는다. 경험이 제공되지 않으면 빈 소제목, 귀가 또는 다음 일정을 만들지 않고 구역 전체를 생략한다.

## 새 교정 추가 절차

1. 사용자의 표현을 지시로 확인하고 적용 시점을 기록한다.
2. 네 필드를 모두 채우되 특정 업체의 사실과 일반 규칙을 분리한다.
3. 기존 규칙과 충돌하면 최신 교정을 우선하고 충돌한 이전 규칙을 명시적으로 갱신한다.
4. 표본 문장이나 눈에 띄는 표현을 교정 예시로 저장하지 않는다.

### CORRECTION-NATURAL-FLOW-01

- 기존 초안의 문제: '도입', '메인 메뉴와 먹는 방법' 등 단조롭고 기계적인 고정 카테고리명을 소제목으로 그대로 노출하여 인공지능 보고서처럼 딱딱하고 어색했다.
- 사용자가 요구한 수정: 기계적인 템플릿 카테고리명을 버리고, 구역의 역할을 담아낸 생동감 있는 서사형/문장형 소제목과 유기적인 스토리텔링으로 글을 풀어나간다.
- 재사용 가능한 일반 규칙: H2 소제목은 기계적 라벨(도입, 장소 정보, 우리가 고른 메뉴 등)을 그대로 출력하지 않고, 해당 구역의 역할(방문 동기, 주차 팁, 메뉴 고민, 맛/식감, 분위기, 총평 등)을 완벽히 수행하는 자연스러운 서사형 제목으로 가공한다. 본문 전개는 단순 팩트 나열이 아니라 실제 사용자의 동선과 호흡에 맞춘 스토리텔링으로 전개한다.
- 특정 글에만 해당하는 예외: 없음.

### CORRECTION-VOICE-02 (2026-07-23)

- 기존 초안의 문제: 사용자의 기존 글에서 보이던 인사말, 구어체 종결, 이모지, 웃음 표현과 방문자의 호흡이 사라져 보고서처럼 읽혔다.
- 사용자가 요구한 수정: 기존 글의 말투를 살려 먼저 원고를 쓰고, 사용자 확인 뒤 스킬에 누적한다.
- 재사용 가능한 일반 규칙: 사용자 예시 글의 말투를 최우선 스타일 자료로 삼고, 정보 전달 문장도 `~요`, `~더라고요`, `ㅋㅋㅋ`, `ㅎㅎ`, 이모지 등 실제 리듬을 사실 범위 안에서 유지한다.
- 특정 글에만 해당하는 예외: 사용자가 별도 격식체를 요청하면 최신 요청을 따른다.

### CORRECTION-VENUE-INFO-01 (2026-07-23)

- 기존 초안의 문제: 사용자가 처음 제공한 가게 정보와 장소 정보 블록이 본문에서 축약되거나 빠졌다.
- 사용자가 요구한 수정: 가게 정보, 편의시설, 주차, 운영 원칙, 생삼겹살 안내를 장소 정보 구역에 그대로 반영한다.
- 재사용 가능한 일반 규칙: 사용자 제공 가게 정보는 1차 근거로 보존하고, `📌 방문 정보` 블록으로 상호·위치·운영 정보·편의시설·주차를 분리한다. 자료에 없는 주소·영업시간·전화번호는 만들지 않는다.
- 특정 글에만 해당하는 예외: 첨부 자료의 상호가 대상 업체와 다르면 해당 자료의 정보를 대상 글에 이식하지 않는다.

### CORRECTION-FACT-BOUNDARY-01 (2026-07-23)

- 기존 초안의 문제: 주차 메모를 별도 주차 옵션처럼 확정해 표현했고, 사용자가 제외하라고 한 항목을 원고에 남겼다.
- 사용자가 요구한 수정: 최신 피드백에 따라 확인된 주차 정보만 남기고, 의심되거나 제외된 항목은 원고·근거 문서에서 함께 삭제한다.
- 재사용 가능한 일반 규칙: 최신 사용자 정정은 이전 메모보다 우선하며, 정정된 사실은 `post.md`, `context.md`, `source_pack.json`에 동시에 반영한다.
- 특정 글에만 해당하는 예외: 없음.

### CORRECTION-HTML-BREAK-01 (2026-07-23)

- 기존 초안의 문제: HTML에서 `<br>`로만 줄바꿈을 처리해 네이버 스마트에디터 붙여넣기 후 문장이 붙었다.
- 사용자가 요구한 수정: 의미 단위 줄바꿈은 유지하고, 문단 사이에는 두 줄 공백을 넣는다.
- 재사용 가능한 일반 규칙: 컴파일러는 의미 단위마다 독립 `<div>`를 만들고, 마크다운 빈 줄마다 `<div>&nbsp;</div>` 두 개를 출력한다. 본문에는 `<br>`를 사용하지 않는다.
- 특정 글에만 해당하는 예외: 없음.
