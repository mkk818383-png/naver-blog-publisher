# 모바일 줄바꿈 가이드

이 문서는 다온로그 본문의 모바일 줄바꿈 계약을 정한다. 아래 수치는 표본에서 계산한 통계가 아니라 사용자가 승인한 규범이다.

## 작성 순서

먼저 문장을 자연스럽게 끝까지 쓴다. 그다음 뜻이 이어지는 덩어리를 살피며 줄을 나눈다. 공백을 제외한 13자를 목표로 삼되, 숫자를 맞추려고 어순을 비틀거나 뜻을 훼손하지 않는다.

조사와 앞말, 사람이나 상품의 이름, 숫자와 단위는 한 덩어리로 읽는다. 이런 요소를 억지로 떼어 내야만 길이를 맞출 수 있다면 문장 전체를 다시 다듬거나 해당 줄을 허용 범위 안에서 길게 둔다.

## 자동 검사 대상

검사기는 마크다운 전체 문법이 아니라 줄 단위의 제한된 문법을 사용한다. 아래 조건을 모두 만족하는 줄만 `eligible` 일반 본문 줄이다.

- 공백을 제거한 뒤 내용이 있다.
- H1 `# ` 또는 H2 `## `로 시작하는 제목이 아니다.
- 주석, 코드 펜스 또는 인용문으로 무시되는 줄이 아니다.
- 아래 제외 문법에 해당하지 않는다.

H3 `### `부터는 제목으로 인식하지 않으므로 다른 제외 조건이 없다면 일반 본문 줄로 계산된다.

### 무시되는 줄

- `<!--`로 시작하는 줄부터 `-->`가 나타나는 줄까지의 HTML 주석
- ` ``` ` 또는 `~~~`로 시작하는 펜스 표식과 그 안의 줄
- 앞 공백을 제거한 뒤 `>`로 시작하는 인용문

### 제외되는 줄

- 빈 줄
- `-`, `+`, `*` 또는 `1.`, `1)`처럼 숫자와 구분 기호 뒤에 공백이 오는 목록 줄
- `|`로 시작하는 표 줄
- `![[`, `![`, `[지도:`로 시작하는 미디어 또는 지도 줄
- `<`로 시작하는 HTML 줄
- `*내돈내산` 또는 `*본 포스팅`으로 시작하는 고지 줄
- `📍`, `⏰`, `🚘`, `☎`, `☎️`, `주소:`, `영업시간:`, `전화번호:`, `주차:`로 시작하는 정보 줄
- `내돈내산`, `제공받아`, `협찬`, `원고료` 중 하나를 어디에든 포함하는 고지 줄

접두어 판정은 줄 양끝 공백을 제거한 값에 적용된다. 위 문법과 다르게 쓴 캡션이나 정보 줄은 자동으로 제외된다고 가정하지 않는다.

## 길이 계산과 통과 조건

각 일반 본문 줄에서 모든 유니코드 공백 문자를 제거한 뒤 Python 문자열 길이, 즉 유니코드 코드 포인트 수를 센다. 화면상의 글자 모양 수, 바이트 수, 한글 음절 폭을 재는 방식이 아니다.

- 목표: 공백 제외 13자
- 범위: 일반 본문 줄의 80% 이상이 공백 제외 10~16자
- 최대: 모든 일반 본문 줄이 공백 제외 20자 이하
- 연속 단문: 공백 제외 9자 이하인 일반 본문 줄 두 개가 바로 이어지면 실패
- 빈 집합: 일반 본문 줄이 하나도 없으면 실패

80% 조건은 정수 반올림을 사용하지 않는다. 범위 안 줄 수에 100을 곱한 값이 전체 일반 본문 줄 수에 80을 곱한 값 이상이어야 한다.

연속 단문 판정에서는 원본의 바로 다음 줄만 비교한다. 두 짧은 일반 본문 줄 사이에 빈 줄, 제목, 목록, 표, 정보, 미디어, 지도, 고지, 주석, 펜스 또는 인용문처럼 대상이 아닌 줄이 하나라도 있으면 연속 관계가 끊긴다.

## 의미 단위 수동 점검

자동 검사 통과는 의미가 자연스럽게 나뉘었다는 증명이 아니다. 최종 원고는 아래 항목을 사람이 직접 읽고 확인한다.

- 조사만 다음 줄로 떨어지지 않았는가
- 고유명사의 중간을 끊지 않았는가
- `250ml`, `3인분`처럼 숫자와 단위를 갈라놓지 않았는가
- 앞줄과 뒷줄이 각각 자연스러운 의미 덩어리인가
- 짧은 줄을 연달아 두어 호흡이 불필요하게 끊기지 않았는가

### 새로 작성한 예시

자연스러운 분리:

```text
창가 쪽 자리를 골라서
천천히 이야기를 나눴어요
```

피해야 할 분리:

```text
따뜻한 차를 친구와
마셨어요
```

두 번째 예시는 조사와 서술 흐름이 어색하게 갈라진다. `친구와 따뜻한 차를 마셨어요`처럼 문장을 먼저 고친 뒤 뜻이 유지되는 위치를 다시 찾는다.

## HTML 내보내기 계약

마크다운의 줄바꿈과 문단 공백은 네이버 스마트에디터에 붙여넣은 뒤에도 구분되어야 한다.

- 의미 단위 줄: `<div>문장</div>`
- 문단 사이의 빈 줄 하나: `<div>&nbsp;</div>` 두 개
- 본문 줄바꿈용 `<br>` 사용 금지

`<br>`를 한 문단 안에 넣는 방식은 스마트에디터 붙여넣기 과정에서 문장들이 합쳐질 수 있다. 컴파일러 회귀 테스트는 의미 단위 줄이 독립 블록인지와 문단 공백 블록이 두 개인지를 함께 확인한다.
