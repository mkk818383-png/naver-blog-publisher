# 다온로그 스타일 가이드

이 문서는 맛집 방문기의 정보 배치와 서술 태도를 정한다. 표본은 역할별로만 참고하며, 표본의 문장이나 서로 다른 전체 구성을 재사용하지 않는다.

## 규칙 우선순위

충돌할 때는 아래 순서로 판단한다.

1. 사용자의 최신 수정 요청
2. 누적된 사용자 수정 이력
3. 승인된 고정 다온로그 템플릿
4. 아래 세 핵심 표본의 역할별 스타일
5. 일반적인 네이버 블로그 관행

현재 호칭은 `남친`이다. 표본에 다른 호칭이 있어도 `예랑`, `예비신랑`은 사용하지 않는다.

## 승인된 고정 구조

아래 순서와 구역 수는 표본의 공통점을 추정한 결과가 아니라 사용자가 승인한 규범이다.

1. 도입
2. 장소 정보
3. 주차와 찾아가는 길
4. 메뉴판 소개
5. 우리가 고른 메뉴
6. 밑반찬과 기본 상차림
7. 가게 내부와 공간 이야기
8. 메인 메뉴와 먹는 방법
9. 식사 후 일정: 사용자가 실제 식사 후 경험을 제공한 경우에만 포함하며, 그 외에는 빈 제목 없이 생략
10. 총평

확인되지 않은 세부 정보나 경험은 만들지 않는다. 선택 구역을 생략해도 나머지 순서는 바꾸지 않는다.

## 표본별 제한된 역할

### 푸른등

- `STYLE-BLUE-01` 방문 동기를 업체 설명보다 앞에 두고, 실제 일정과 선택 이유로 시작한다.
- `STYLE-BLUE-02` 주문 전후와 상차림, 식사, 추가 주문의 실제 시간 흐름을 이어 쓴다.
- `STYLE-BLUE-03` 막연한 단점 대신 무엇이 부족했고 당시 어떻게 대응했는지 구체적으로 적는다.
- 이 표본은 메뉴 탐색, 장소 안내, 고정 구역 순서의 근거로 확대하지 않는다.

### 류센소

- `STYLE-RYU-01` 메뉴판에서 실제로 비교하거나 고민한 선택지를 설명한다.
- `STYLE-RYU-02` 주문한 메뉴와 선택 결과를 분명히 연결한다.
- `STYLE-RYU-03` 직접 먹은 음식은 메뉴별로 맛, 향, 식감처럼 확인된 감각을 구분해 적는다.
- `STYLE-RYU-04` 가게가 안내한 방법과 실제로 시도한 먹는 순서를 구분해 설명한다.
- 이 표본은 방문 동기, 주차 동선, 고정 구역 순서의 근거로 확대하지 않는다.

### 물총칼국수

- `STYLE-WATER-01` 위치는 주변 기준점과 방문 전후 동선을 이용해 이해하기 쉽게 설명한다.
- `STYLE-WATER-02` 주차는 실제 이용 여부와 공간, 제한 시간, 비용 등 확인된 조건을 나눈다.
- `STYLE-WATER-03` 대기, 결제, 좌석, 동반 조건 등 이용 안내는 방문 판단에 필요한 정보만 정리한다.
- `STYLE-WATER-04` 주차 또는 이동부터 입장까지 실제 순서대로 연결한다.
- 이 표본은 음식 평가, 개인적 아쉬움, 고정 구역 순서의 근거로 확대하지 않는다.

## 공통 서술 계약

- `STYLE-COMMON-01` 장소 사실과 사용자 체험을 분리한다. 외부 자료의 경험을 사용자의 경험으로 바꾸지 않는다.
- `STYLE-COMMON-02` 사용자가 제공한 일정, 동행, 선택 이유가 있을 때 개인 맥락을 먼저 살린다.
- `STYLE-COMMON-03` 개인적인 추억과 농담은 입력에 있는 내용만 사용한다.
- `STYLE-COMMON-04` 칭찬만 나열하지 않고 실제로 겪은 구체적인 아쉬움이 있으면 총평에 반영한다.
- `STYLE-COMMON-05` 문장을 먼저 자연스럽게 완성한 뒤 의미 단위로 줄을 나눈다. 길이 수치는 별도의 승인된 줄바꿈 계약을 따른다.

## 금지 사항

- 표본의 눈에 띄는 문장, 비유, 감탄 표현을 복제하거나 표현 목록으로 만들지 않는다.
- 세 표본의 서로 다른 전체 구성을 평균내거나 새 템플릿으로 만들지 않는다.
- 표본에 한 번 나온 사실이나 경험을 다른 업체 글에 옮기지 않는다.
- 자료에 없는 방문, 주문, 맛, 감정, 불편을 추정하지 않는다.
